scrapy 框架介绍与安装

scrapy 框架介绍

Scrapy是一个为了爬取网站数据，提取结构性数据而编写的应用框架，我们只需要实现少量的代码，就能够快速的抓取。

Scrapy 使用了Twisted['twɪstɪd]异步网络框架，可以加快我们的下载速度。

文档地址：http://scrapy-chs.readthedocs.io/zh_CN/1.0/intro/overview.html

scrapy 安装

注意： 使用虚拟环境再安装引擎

pip install scrapy

安装完成后在命令行，如果信息显示说明安装成功

scrapy --help

scrapy 快速体验

使用流程

  1. 创建项目
    scrapy startproject 项目名称
  2. 创建爬虫
    cd 项目目录
scrapy genspider 爬虫名称 允许域名

  3. 编写爬虫组件（解析数据）和管道组件（保存数据）

    # -*- coding: utf-8 -*-
import scrapy
class ItSpider(scrapy.Spider):
   name = 'it'
   allowed_domains = ['itcast.cn']
   # 输入首个爬取网址
   start_urls = ['http://www.itcast.cn/channel/teacher.shtml#ajavaee']

   def parse(self, response):
       print(response.xpath('//div[@class="li_txt"]/h3/text()').extract())
       pass
  4. 启动爬虫
    scrapy crawl 爬虫名称

scrapy 爬虫组件-提交数据

作用

实现提取数据和提取请求，并且提交给引擎。

实现流程

  1. 继承自爬虫类
    class ItcastSpider(scrapy.Spider):
 ...
  2. 定义爬虫名称
    name = "itcast"
  3. 设置允许爬取的范围
    allowed_domains = ['itcast.cn']
  4. 设置开始爬取的请求地址
    start_urls=['http://www.itcast.cn/channel/teacher.shtml']
  5. 实现解析函数，提取数据或者提取URL，并且提交给引擎
    def parse(self,response):
 print(response.xpath('//div[@class="li_txt"]/h3').extract())

解析函数对象详解

response 对象

封装从互联网上返回的响应

  • response.status 响应状态码
  • response.headers 响应头
  • response.text 响应内容
  • response.request.cookies 获取cookies
Selector 对象

封装数据操作的对象

  • xpath 可以使用 xpath 提取数据，返回 SelectorList 对象
  • css 可以使用 css 样式选择器提取数据，返回 SelectorList 对象
  • extract 从selector对象中提取内容
SelectorList 对象

返回 Selector 对象的合集

  • extract_first 取列表中第一个元素的内容，如果元素不存在返回 None
  • extract 提取列表中selector对象中所有内容

提交数据给引擎

使用关键词 yield，在爬虫组件的解析函数中提交数据给引擎

def parse(self,response):
  ...
  item = {}
  item["data"] = "数据"
  yield item
提交数据类型
  • BaseItem
  • Request
  • dict
  • None

scrapy 管道组件-保存数据

作用

专门用于保存数据

系统管道

在运行爬虫时添加 -o 数据文件

scrapy crawl 爬虫名称 -o 导出文件

自定义管道

多管道运行流程

上一个管道的数据给下一个管道使用

实现流程
  • 在 pipelines.py 中编写管道实现
    1. 定义管道类
    2. 必须实现 process_item(self,item,spider) 在管道接收数据对象时回调，用于接收和处理每个数据对象
    3. 可选实现
      • open_spider(self,spider) 在爬虫启动时回调，主要用于一些初始化操作
      • close_spider(self,spider) 在爬虫关闭时回调，主要用于一些释放操作
  • 在 settings.py 中开启管道
    1. 参数的值是一个权重，越小越优先执行
注意事项
  1. 在管道实现函数中process_item必须有返回值，传递给下一个管道
  2. 多管道实现时可以通过类型判断或爬虫名称判断区分存储的数据

scrapy 数据模型对象

作用

防止在爬取数据过程中写错

实现流程

  1. 在 items.py 中定义模型对象
    class ItcastItem(scrapy.Item):
 name = scrapy.Field()
 pass

  2. 在爬虫组件中使用数据模型

    from myspider.items import ItcastItem

class ItSpider():
 def parse(self,response):
   item = ItcastItem()
   item["name"] = "xxxx"
   yield item

scrapy 管道组件-提交请求

实现多请求方式

  • 在解析函数中继续请求下一页
  • 在 start_urls 实现多请求

scrapy.Request 构建方式

scrapy.Request(url[,callback,method="GET",headers,body,cookies,meta,dont_filter=False])
  • callback 表示当前的url的响应交给哪个函数去处理
  • meta 实现数据在不同的解析函数中传递，meta默认带有部分数据，比如下载延迟，请求深度等
  • dont_filter 默认会过滤请求的url地址，即请求过的url地址不会继续被请求，对需要重复请求的url地址可以把它设置为Ture，比如贴吧的翻页请求，页面的数据总是在变化;start_urls中的地址会被反复请求，否则程序不会启动

提交请求给引擎

使用关键词 yield

def parse(self,response):
  ...
  yeild scrapy.Request(
    url='xxx',
    callback='解析函数'
  )
