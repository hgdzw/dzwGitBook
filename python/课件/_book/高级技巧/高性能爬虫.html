线程回顾与守护线程

线程知识点回顾

# 创建线程对象
t1 = threading.Thread(target=func,args=(,))
# 设置为守护线程
t1.setDaemon(True)
# 启动线程
t1.start()

注意：

非守护线程：在python3中，当主线程执行结束，程序不会自动结束，只有在所有线程执行结束时才会结束。

守护线程： 如果设置子线程为守护线程,那么当主线程结束时不管守护线程运行到任何状态时都会自动结束。

队列知识点

from queue import Queue
# 构建队列
q = Queue(maxsize=100)

item = {}
# 把数据放入到队列中，如果队列数据已满，就进行等待直到队列不满时把数据添加到队列中
q.put(item)
# 把数据放入到队列中，如果队列数据已满，就不进行等待，引发异常
q.put_nowait(item)  

# 把数据从队列中取出，如果队列为空，就进行等待直到队列不为空，返回数据
q.get()
# 把数据从队列中取出，如果队列为空，就不进行等待，引发异常
q.get_nowait()

# 获取队列中元素个数
q.qsize()
# 获取未完成的任务数
q.unfinished_tasks

# 让 unfinished_tasks 属性 -1
q.task_done()

# 当前程序进行等待状态，直到队列的 unfinished_tasks 属性为 0 时继续执行
q.join()

注意：

  • 当调用 put 或 put_nowait 方法时 q.qsize() 会 + 1，并且 q.unfinished_tasks 也会 + 1
  • 当调用 get 或 get_nowait 方法时 q.qsize() 会 -1，但是 unfinished_tasks 不发生变化
  • 当调用 task_done 方法时 unfinished_tasks 会 -1

线程池

问题

计算机每次创建线程和销毁线程需要额外占用资源，频繁创建和销毁线程会导致计算机性能下降。

解决思路

使用池化技术，预先常见一定的线程数量放到池中，需要执行代码时从池中获取创建好的线程，让该线程执行代码任务，执行完成后把线程放回池中，给其他代码任务代码使用。

优点

  1. 减少因为创建线程和销毁线程带来不必要的性能损耗
  2. 让程序更加稳定，不会因为同一时间创建线程过多而导致内存不够使程序引发系统一系列的问题。

实现流程

  1. 导入线程池模块
    from multiprocessing.dummy import Pool
  2. 创建线程池
    self.pool = Pool(5)
  3. 实现任务执行代码
     def exec_task(self):
     print("任务执行代码")
     return "执行任务返回结果"
  4. 实现任务完成回调
     def exec_task_finish(self,result):
     print("执行任务完成回调函数")
    注意： 必须要有 result 参数，result 参数表示执行任务代码的返回值
  5. 让任务代码在线程池中的线程上执行
    self.pool.apply_async(self.exec_task,callback=self.exec_task_finish)

实现代码

#!/usr/bin/python3
# -*- coding: utf-8 -*-

# 1. 导入线程池模块
from multiprocessing.dummy import Pool

class Spider():
    def __init__(self):
        # 2. 创建线程池对象，参数为初始化多少个线程
        self.pool = Pool(5)
        pass

    # 3. 定义执行任务代码
    def exec_task(self):
        print("任务执行代码")
        return "执行任务返回结果"

    # 4. 实现任务完成回调
    def exec_task_finish(self,result):
        print("执行任务完成回调函数")

    def run(self):
        # 5. 让任务代码在线程池中的线程执行
        self.pool.apply_async(self.exec_task,callback=self.exec_task_finish)
        pass

if __name__ == '__main__':
    spider = Spider()
    spider.run()

页面地址: http://www.qiushibaike.com/8hr/page/1/

    • url地址的规律非常明显，一共只有13页url地址

实现流程

代码实现
#!/usr/bin/python3
# -*- coding: utf-8 -*-

import requests
from lxml import etree

class QiushiSpider():

    def __init__(self):
        self.base_url = "http://www.qiushibaike.com/8hr/page/{}"
        self.headers = {
            "User-Agent":"Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/71.0.3578.98 Safari/537.36"
        }
        pass

    def get_url_list(self):
        '''
        获取 URL 列表
        :return: 
        '''
        return [self.base_url.format(page) for page in range(1,14,1)]

    def get_html(self,url):
        '''
        发送请求，获取 HTML
        :param url: 
        :return: 
        '''
        response = requests.get(url,headers=self.headers)
        return response.text

    def get_items(self,html):
        '''
        提取数据
        :param html: 
        :return: 
        '''
        eroot = etree.HTML(html)
        return eroot.xpath('//a[@class="recmd-content"]/text()')

    def save_item(self,item):
        print(item)

    def run(self):
        # 1. 获取 url
        url_list = self.get_url_list()

        for url in url_list:
            # 2. 发送请求获取 html
            html = self.get_html(url)

            # 3. 提取数据
            items = self.get_items(html)

            # 4. 保存数据
            for item in items:
                self.save_item(item)
        pass

if __name__ == '__main__':
    spider = QiushiSpider()
    spider.run()

案例二、糗事百科多线程

多线程实现设计思路

生产消费模式

标准爬虫流程

多线程实现流程

多线程实现流程

  1. 创建数据容器对象
    1. URL 容器
    2. HTML 容器
    3. 数据 容器
  2. 实现处理任务函数
    1. 实现生产 URL 任务
    2. 实现消费 URL 生产 HTML 任务
    3. 实现消费 HTML 生产 数据 任务
    4. 实现消费数据 任务
  3. 让任务函数运行在线程上，并且以守护线程的形式运行
  4. 主线程退出条件
  5. 调整线程数量实现并行执行

多线程实现代码

#!/usr/bin/python3
# -*- coding: utf-8 -*-


from queue import Queue
import requests
from lxml import etree
from threading import Thread as Task

class QiushiSpider():

    def __init__(self):
        self.base_url = "http://www.qiushibaike.com/8hr/page/{}"
        self.headers = {
            "User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/71.0.3578.98 Safari/537.36"
        }

        # 创建 url 存储容器
        self.url_queue = Queue()

        # 创建 html 存储容器
        self.html_queue = Queue()

        # 创建 数据 存储容器
        self.item_queue = Queue()

        pass

    def get_url_list_task(self):
        '''
        实现生产 URL 任务
        :return: 
        '''
        for page in range(1,14,1):
            self.url_queue.put(self.base_url.format(page))

    def get_html_task(self):
        '''
        实现消费 URL 生产 HTML 任务
        :return: 
        '''
        while True:
            url = self.url_queue.get()

            response = requests.get(url,headers=self.headers)

            html = response.text

            self.html_queue.put(html)

            # 通知 URL 队列完成任务
            self.url_queue.task_done()

    def get_item_task(self):
        '''
        实现消费 HTML 生产 数据 任务
        :return: 
        '''
        while True:
            html = self.html_queue.get()
            eroot = etree.HTML(html)
            titles = eroot.xpath('//a[@class="recmd-content"]/text()')
            for title in titles:
                self.item_queue.put(title)

            # 通知 HTML 队列完成任务
            self.html_queue.task_done()

    def save_item_task(self):
        '''
        消费数据
        :return: 
        '''
        while True:
            item = self.item_queue.get()

            print(item)

            # 通知 数据 队列完成任务
            self.item_queue.task_done()


    def run(self):

        # 定义所有的任务列表
        tasks = []

        # 生产 URL 任务 放在 子线程上运行
        get_url_list_task = Task(target=self.get_url_list_task)
        tasks.append(get_url_list_task)

        # 消费 URL 生产 HTML 任务 放在子线程上运行
        for i in range(5):
            get_html_task = Task(target=self.get_html_task)
            tasks.append(get_html_task)

        # 消费 HTML 生产 数据 任务 放在子线程上运行
        for i in range(2):
            get_item_task = Task(target=self.get_item_task)
            tasks.append(get_item_task)

        # 消费数据的任务 放在子线程上运行
        save_item_task = Task(target=self.save_item_task)
        tasks.append(save_item_task)

        # 让所有子线程启动并且以守护线程运行
        for task in tasks:
            task.setDaemon(True)
            task.start()

        # 保证子线程必须运行
        time.sleep(2)

        # 设置主线程退出条件
        for queue in [self.url_queue,self.html_queue,self.item_queue]:
            queue.join()

        pass

if __name__ == '__main__':
    spider = QiushiSpider()
    spider.run()

案例三、糗事百科多进程

实现思路

线程改成进程

由于整体思路不变，仅仅只是原来运行在线程上的代码需要运行在进程上所以仅需要线程修改成进程即可

# from threading import Thread as Task
from multiprocessing import Process as Task
Queue 改成 JoinableQueue

由于进程间无法共享一个变量，所以需要把队列改编成 JoinableQueue

JoinableQueue 队列上层使用 Queue 相同的接口，底层使用进程间通讯。

# from queue import Queue
from multiprocessing import JoinableQueue as Queue
设置守护进程

守护线程和守护进程的设置方式不同

# task.setDaemon(True)
task.daemon = True

案例四、糗事百科线程池

流程图

实现流程

  1. 导入线程池模块
  2. 创建线程池对象
  3. 实现定义任务代码
    1. 从 url 队列中获取 url
    2. 发送请求获取 html
    3. 从 html 中提取数据
    4. 保存数据
  4. 实现任务完成回调
    1. 在线程池中的线程上继续执行任务代码
  5. 在线程池中的线程上执行任务代码
    1. 循环开启多个任务同时执行

代码实现

#!/usr/bin/python3
# -*- coding: utf-8 -*-
# 1. 导入线程池模块
# 进程池
# from multiprocessing import Pool
# 线程池
from multiprocessing.dummy import Pool
import time
from queue import Queue

import requests
from lxml import etree

class QiushiSpider():
    def __init__(self):

        # 2. 创建线程池，初始化线程数量
        self.pool = Pool(5)

        self.base_url = "http://www.qiushibaike.com/8hr/page/{}/"
        self.headers = {
            "User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/71.0.3578.98 Safari/537.36"
        }

        # 专门存放 url 容器
        self.url_queue = Queue()
        pass

    def get_url_list(self):
        '''
        获取 url 列表放入到 url 容器中
        :return: 
        '''
        for page in range(1,14,1):
            url = self.base_url.format(page)
            self.url_queue.put(url)


    # 3. 实现执行任务
    def exec_task(self):
        # 1> 获取url
        url = self.url_queue.get()
        # 2> 发送请求获取 html
        response = requests.get(url,headers=self.headers)
        html = response.text
        # 3> 解析 html 提取数据
        eroot = etree.HTML(html)
        titles = eroot.xpath('//a[@class="recmd-content"]/text()')
        for title in titles:
            item = {}
            item["title"] = title

            # 4> 保存数据
            print(item)
        self.url_queue.task_done()

    # 4. 实现执行任务完成后的操作,必须至少有一个参数
    # result 任务执行的最终结果的返回值
    def exec_task_finished(self,result):
        print("result:",result)
        print("执行任务完成")
        self.pool.apply_async(self.exec_task,callback=self.exec_task_finished)

    def run(self):
        self.get_url_list()
        # 5. 让任务使用线程池中的线程执行并且设置执行后的回调操作
        # callback 表示执行完成后的回调
        for i in range(5):
            self.pool.apply_async(self.exec_task,callback=self.exec_task_finished)

        self.url_queue.join()
        pass

if __name__ == '__main__':
    spider = QiushiSpider()
    spider.run()

总结

  • 守护线程与非守护线程
    • 非守护线程 当所有线程执行完成程序才退出
    • 守护线程 不管子线程运行到任何状态，主线程执行完成程序退出子线程直接结束
  • 队列
    • join 主线程进行等待直到队列为空（unfinished_tasks = 0）
  • 多线程与多进程实现思路
    • 边生产边消费设计方案
  • 什么是线程池
    • 预先创建好线程并且放在池中，需要执行任务代码就从池中获取线程执行任务代码
  • 线程池优点
    • 减少因为创建线程和销毁线程带来不必要的性能损耗
    • 让程序更加稳定，不会因为同一时间创建线程过多而导致内存不够使程序引发系统一系列的问题。
  • 线程池使用流程
    • 导入线程池模块
    • 创建线程池对象
    • 定义任务执行代码函数
    • 定义任务执行完成代码函数
    • 让任务代码函数在线程池中的线程上执行
  • 线程池与协程池实现思路
    • 让任务代码执行在线程池中的线程上执行
    • 让任务代码执行在协程池中的执行
